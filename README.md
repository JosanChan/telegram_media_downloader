## 指令对比速查

| 指令 | 适用场景 | 处理粒度 | 需要讨论组 |
|------|----------|----------|-----------|
| `/forward_screenshot` | 单条视频 → 封面图帖 + 评论区视频 | 逐条消息独立处理 | 是 |
| `/forward_multi` | 批量转发，每条/每组独立生成帖子 | 逐条消息/逐媒体组独立处理 | 是 |
| `/forward_album` | 批量视频 → 合并为媒体组 | 全部消息合并为一批 | 否 |
| `/forward_clone` | 克隆"封面图+评论区视频"结构的频道 | 逐条消息独立处理 | 是 |

## 新增指令详细说明

所有指令均通过 Bot 发送，格式统一为：

```
/指令名 源频道链接 目标频道链接 起始消息ID 结束消息ID [过滤条件]
```

---

### 1. `/forward_screenshot` — 单视频截图转发

**功能**

将范围内的每条**单视频消息**逐一以"封面图发主帖、视频发评论区"的形式转发到目标频道。每条视频生成一个独立帖子。

**消息类型处理规则**

| 原始消息类型 | 处理方式 |
|------------|---------|
| 单条视频 | 缩略图 → 目标频道主帖；视频 → 评论区 |
| 单条图片 / 文字 / 文档 | 普通转发（直接复制到目标频道） |
| 媒体组（多条合并的消息） | **跳过**；任务结束后发送一条反馈，列出所有被跳过的消息 ID |

> 媒体组需改用 `/forward_multi` 处理。

**使用方法**

```
/forward_screenshot https://t.me/源频道 https://t.me/目标频道 起始ID 结束ID
```

**示例**

```
/forward_screenshot https://t.me/src_channel https://t.me/dst_channel 100 200
```

> 注意：目标频道必须开启讨论组（linked discussion group），否则视频无处投放。

### 2. `/forward_multi` — 批量截图转发

**功能**

批量转发，每条消息 / 每个媒体组**独立**生成一个帖子。支持两种子模式，发送命令后通过弹出的内联按钮选择。

**子模式说明**

| 模式 | 按钮文字 | 区别 |
|------|---------|------|
| 单缩略图（Single） | 📷 单个缩略图 + 合并评论区 | 每组只生成 1 张主帖封面图 |
| 多缩略图（Multi） | 📷📷 多个缩略图(相册) + 评论区 | 每组生成包含所有封面图的相册主帖 |

**各消息类型的处理规则**

| 原始消息类型 | 单缩略图模式 | 多缩略图模式 |
|------------|------------|------------|
| **单条视频** | 缩略图 → 主帖；视频 → 评论区 | 同左 |
| **单条图片 / 文字 / 文档** | 普通转发 | 普通转发 |
| **媒体组 — 纯视频** | 第一个视频的缩略图 → 主帖；全部视频 → 评论区 | 全部视频缩略图组成相册 → 主帖；全部视频 → 评论区 |
| **媒体组 — 图片 + 视频** | 看第一项：图片则用图片，视频则用缩略图 → 主帖；全部图片 + 全部视频 → 评论区 | 全部图片 + 全部视频缩略图组成相册 → 主帖；仅视频 → 评论区（图片不进评论区） |
| **媒体组 — 纯图片** | 第一张图片 → 主帖；全部图片 → 评论区 | 全部图片组成相册 → 主帖；评论区无内容 |

> **媒体组的独立性**：原频道的每个媒体组（帖子）在目标频道生成一个独立的帖子，不会跨组合并。5 条视频若是同一个媒体组，目标频道产生 1 个帖子；若是 5 条独立消息，则产生 5 个帖子。

**使用方法**

```
/forward_multi https://t.me/源频道 https://t.me/目标频道 起始ID 结束ID
```

发送命令后，Bot 弹出模式选择按钮，点击即开始处理。

**示例**

```
/forward_multi https://t.me/src_channel https://t.me/dst_channel 100 200
```

> 注意：不支持源频道开启了"转发限制"（protected content）的情况。

### 3. `/forward_album` — 多视频合并媒体组转发

**功能**

将范围内的多条视频消息打包合并为 Telegram 媒体组（Album）发送到目标频道，每 10 条为一组，最终呈现为 1 个或若干个媒体组帖子。

**效果**

- 目标频道：每 10 条视频生成一个媒体组帖子（包含原始视频文件）
- 保留原始视频和文案

**使用方法**

```
/forward_album https://t.me/源频道 https://t.me/目标频道 起始ID 结束ID
```

**示例**

```
/forward_album https://t.me/src_channel https://t.me/dst_channel 100 110
```

> 注意：不支持源频道开启了"转发限制"的情况。

### 4. `/forward_clone` — 克隆频道（保留讨论区结构）

**功能**

将源频道的帖子**连同其讨论区评论**一起复制到目标频道，完整还原"封面图主帖 + 评论区视频"结构。

**处理逻辑**

1. 逐条读取源频道指定范围内的帖子
2. 将主帖（图片、视频、文字等）复制到目标频道
3. 读取该帖在源频道讨论组中的所有评论
4. 将这些评论复制到目标频道的讨论组，作为对应新帖的回复

**效果**

- 源频道：`[图片帖]` → 评论区：`[视频1]` `[视频2]`
- 目标频道：`[图片帖（复制）]` → 评论区：`[视频1（复制）]` `[视频2（复制）]`

**使用方法**

```
/forward_clone https://t.me/源频道 https://t.me/目标频道 起始ID 结束ID
```

**示例**

```
/forward_clone https://t.me/src_channel https://t.me/dst_channel 100 200
```

**前提条件**

- 源频道和目标频道都需要有已关联的讨论组（linked discussion group）
- 源频道不能开启"转发限制"（protected content）
- Bot 需要有目标频道及其讨论组的发送权限

## 通用说明

### 消息 ID 获取方式

在 Telegram Web（或桌面端）打开频道，鼠标悬停在帖子上，URL 末尾的数字即为消息 ID。

### 过滤条件（可选参数）

所有指令均支持在末尾添加过滤条件，语法与原项目 `/forward` 指令一致，例如：

```
/forward_clone https://t.me/src https://t.me/dst 1 500 message_date >= 2024-01-01
```

### 停止任务

发送 `/stop` 可中断当前正在执行的任务。
